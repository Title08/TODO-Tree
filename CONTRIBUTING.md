# Contributing

Thanks for contributing to TODO Tree for Obsidian.

## Development setup

1. Clone the repository.
2. Install dependencies:
   ```bash
   npm install
   ```
3. Build once:
   ```bash
   npm run build
   ```
4. During development:
   ```bash
   npm run dev
   ```

## Pull request process

1. Create a branch for your change.
2. Keep changes focused and include clear descriptions.
3. Ensure the project builds successfully.
4. Open a pull request with:
   - What changed
   - Why it changed
   - Any screenshots (for UI changes)

## Commit style

- Use clear, descriptive commit messages.
- Group related changes in a single commit when possible.

## Reporting bugs

Use the bug report issue template and include:

- Steps to reproduce
- Expected vs actual behavior
- Obsidian version
- Plugin version
- Platform/OS
